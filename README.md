# Investment Portfolio Intelligence Tool

A Streamlit application for multi-asset portfolio analysis, built on institutional investment methodology. Designed to support the kind of quantitative portfolio review work central to investment consulting practice.

---

## What this tool does

The app takes a user-defined equity portfolio and produces a full analytical stack:

- **Portfolio optimisation** across the efficient frontier (Modern Portfolio Theory)
- **Institutional risk metrics**: Sharpe, Sortino, CVaR, VaR, maximum drawdown
- **Brinson-Hood-Beebower performance attribution**: allocation effect, selection effect, interaction
- **Benchmark comparison**: active return, tracking error, information ratio
- **Multi-asset frontier extension**: shows how adding a fixed income allocation shifts the risk/return frontier
- **Monte Carlo simulation & stress testing**: projects portfolio outcomes across thousands of simulated paths under user-defined market scenarios
- **Rebalancing simulation**: models portfolio value under different rebalancing frequencies and transaction cost assumptions

## Methodology

### Modern Portfolio Theory (Markowitz, 1952)
The efficient frontier is generated by Monte Carlo simulation of 5,000 random weight combinations. For each portfolio the tool computes annualised return, annualised volatility, and Sharpe ratio. Three portfolios are highlighted: maximum Sharpe, maximum Sortino, and minimum volatility.

### Risk Metrics
- **Sharpe Ratio**: excess return per unit of total volatility. `(Rp - Rf) / σp`
- **Sortino Ratio**: excess return per unit of *downside* volatility only. More appropriate for investors who do not penalise upside variance.
- **Maximum Drawdown**: largest peak-to-trough decline in cumulative portfolio value over the period.
- **Value at Risk (VaR, 95%)**: the daily loss threshold exceeded on the worst 5% of trading days.
- **Conditional VaR / Expected Shortfall (95%)**: the average loss on those worst 5% of days. Preferred over VaR for tail risk measurement because it is coherent and captures the shape of the loss distribution beyond the threshold.

### Brinson-Hood-Beebower Attribution (1986)
Single-period attribution model decomposing active return (portfolio minus benchmark) into:

```
Allocation Effect  = (wp - wb) × (rb - Rb)
Selection Effect   = wb × (rp - rb)
Interaction Effect = (wp - wb) × (rp - rb)
Active Return      = Allocation + Selection + Interaction
```

Where `wp` = portfolio weight, `wb` = benchmark weight, `rp` = portfolio asset return, `rb` = benchmark asset return, `Rb` = total benchmark return.

### Benchmark Metrics
- **Active Return**: annualised mean of the daily (portfolio return − benchmark return) series.
- **Tracking Error**: annualised standard deviation of the active return series. Measures how closely the portfolio follows its benchmark.
- **Information Ratio**: active return divided by tracking error. A consistent IR above 0.5 is considered evidence of skill; above 1.0 is exceptional (Grinold & Kahn, 1999).

### Multi-Asset Frontier
A bond ETF return series is blended with the equity portfolio return series across a sweep of equity/bond allocations (0% to 100% bonds in 1% increments). The resulting frontier is overlaid on the equity-only scatter to show how fixed income shifts the risk/return trade-off. This reflects standard institutional practice of considering fixed income as a separate asset class within a strategic asset allocation (SAA) framework.

### Monte Carlo Simulation
Portfolio paths are simulated using Geometric Brownian Motion with normally distributed daily returns drawn from the historical mean and volatility of the weighted portfolio. Stress scenarios shift these parameters before simulation:

| Scenario | Return shock | Volatility adjustment |
|---|---|---|
| Market Crash | −30% annualised | +50% volatility |
| High Volatility | None | 2× volatility |
| Interest Rate Shock | −15% annualised | +50% volatility |
| Sector Decline | −20% annualised | +20% volatility |
| Custom | User-defined | User-defined |

Output includes simulated path fan chart, distribution of final values, outcome percentiles, and probability of reaching a user-specified target.

**Limitation**: GBM assumes normally distributed returns. Equity returns exhibit fat tails and negative skew, so the model underestimates the probability of extreme losses. CVaR on the Analysis page provides a more conservative tail risk estimate based on actual historical returns.

### Rebalancing Simulation
The simulation initialises a portfolio at target weights and applies daily price changes. On each rebalance date it calculates the trades required to restore target weights, deducts transaction costs from portfolio value, and reinvests at target weights. The cost sensitivity sweep models total costs across transaction cost rates from 0% to 2%.

## Limitations

- **JSE data only (current version)**: suffers from thin trading and staleness in less liquid counters.
- **Single-period Brinson**: does not account for compounding effects in multi-period attribution (see Bacon, 2008, for geometric linking).
- **Normally distributed returns assumed** in Monte Carlo simulation. Actual equity returns exhibit fat tails and negative skew, so tail loss probabilities are likely understated.
- **Transaction cost model is simplified**: does not include JSE securities transfer tax (0.25% on equity purchases), VAT on brokerage, or market impact for large orders.
- **Historical optimisation**: past returns and correlations are not a reliable guide to future portfolio performance.
